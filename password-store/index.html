Password store

Olen käyttänyt jonkin aikaa salasananhallinta- ohjelmia, jotka pystyvät tallentamaan ja luomaan salasanoja eri käyttäjätileille. Pitkän aikaa olen käyttänyt keepassx- nimistä ohjelmaa.

Vaihdoin tämän jonkin aikaa sitten passwordstore- nimiseen ohjelmaan. Se on “unix filosofia” mielessä tehty salasanaohjelma, joka toimii pääasiassa komentokehotteelta. Itse tykkään käyttää komentokehotteita niin tämä tuntuu kivalta käyttää. Löysin archwiki:stä linkkejä ohjelman sivuille ja hyödyllisiin scripteihin muuntaa olevassa olevista salasanaohjelmista salasanat ja käyttäjätilit tähän ohjelmaan.

Passissa on myös integraatio git:in kanssa. Salasanatietokanta voi olla git versionhallinnassa ja sen voi sitä kautta jakaa muihin tietokoneisiin. Itse jaan sen puhelimeeni password store android sovellukseen. Aluksi tein niin, että kloonasin raspberry pihin salasanatietokannan ja pull/push:asin sinne aina kun tein uusia salasanoja. Tämä toimi aluksi hyvin koska pidin raspberry pi:tä enemmän auki, mutta nykyään se on jäänyt pienemmälle käytölle. Nyt minun pitäisi käynnistää se aina kun haluan synkronoida salasanat puhelimeen. Hiljattain korjasin tämänkin hässäkän synkronoimalla niitä suoraan tietokoneesta puhelimeen. En tehnyt tätä aluksi, koska jos synkronoisin suoraan tietokoneeseen git antaisi virhettä, että ei voitu pushata koska nykyinen branch on checkattuna siellä. Tein niin että kloonasin siitä bare repositorion mun kotikansioon ja laitoin puhelimesta passin synkronoimaan sinne. Nyt voi vain synkronoida puhelimen salasanat ja jos olen tietokoneella niin pullata siitä bare reposta salasanat tietokantaan. Tämän saisi ehkä jollain git hookillakin ratkaistua, mikä suorittaisi aina git checkout -f:n kun pushaan tietokoneelle, mutta en tiedä, ehkä siinä voi käydä pahasti jos olen esimerkiski tietokoneella tehnyt jotain muutoksia salasanoihin ja pushaisin puhelimesta salasanat sinne ja se ylikirjoittaisi mun muutokset tai jotain.

Yksi hyödyllinen ominaisuus mikä pass:issa on ollut, on salasanojen grep:aaminen. Sillä löytää kuinka monelle eri käyttäjälle on kierrättänyt kuinka pitkään samaa salasanaa. Jonkun aikaa sitten vaihdoin monelle sivulle satunnaisemman salasanan siltä varalta että joltain niistä on jokin vuoto ja joku saa mun salasanan ja sähköpostiosoitteen käyttöön.
